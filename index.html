React 与 Vue.js 的对比

  • 这两个框架 都是以组件化的思想进行开发的

  • 模块化：以 JS 开发的角度去分析，更多的是站在后台中应用模块化的思想【Node】

    • 模块化的优点：便于文件管理，增加复用，职责单一

  • 组件化：以 Ul 发的角度，将页面拆分成为 多个互不相关的 小组件；

    • 组件化的优点：易于维护，使用便捷，便于扩展

  • Vue.js 怎么实现组件化：

    • 在vue.js中，通过一个后缀名为.vue的文件实现组件化
    • template 定义组件的结构
    • script 定义组件的行为
    • style 定义组件的样式

  • React 怎么实现 组件化

    • 在react中，直接使用JS这门语言，来实现模块化 所有的组件结构/行为/样式 都通过Js直接实现

React 中核心的概念

  1. 虚拟 DOM

    • 为什么要实现虚拟Dom：因为浏览器中默认实现DOM更新的方式速度很慢，效率低下；so 通过js的方式实现虚拟Dom，提高效率
    • 什么是React中的虚拟Dom ：就是利用JS对象的形式，来模拟一棵真正的DOM树

  2. Diff 算法

    • tree diff ：将新旧两颗DOM树，相同的层级节点，从上到下，逐层进行对比的过程，叫做tree diff。好处：对比次数少，当把整个DOM树每一层梳理完成之后，则就找到了需要更改的元素！
    • component diff：在对比每一层的时候，对每一层中的组件进行对比，如果组件类型相同，则暂时认为不需要修改，如果类型不相同，则删除旧组件，创建新组建，将新组建替换到旧组件的位置！
    • element diff：当组件类型相同的时候，需要对组件中的每一个元素进行对比！
    • 在通过 for 循环，循环创建类型相同的组件或者元素的时候，需为每一个创建的组件或元素，添加一个唯一的key值，这样能提高虚拟DOM的对比率

ES6

let

  1. let 的核心是加入块级作用域，{} 就是一个快
  2. 变量声明，必须先声明后使用，不允许重复声明；

const

  1. 声明常量，一旦声明，不可改变,否则报错；
  2. 普通数据类型不能修改，引用类型的数据可以修改属性值
  3. 声明时必须赋值，也是块级作用域，没有变量提升，不可重复声明；

总结

  1. 凡是变量使用 let
  2. 凡是不变的都是使用 const
  • 如果你定义的成员，一次都没有改变过，使用 const
  • 如果你定义的成员，最少做了一次修改，则一定是let
  • 所有相对来说，const使用要多一些；
  • 最佳使用建议：全部 const，报错时才改为let

数组的解构赋值

数组的解构赋值是按照数据的顺序来解构的

对象的解构赋值

对象的解构赋值 是按照对象的 key 来解构的；当解构不成功的时候，默认赋值undefined

运用场景

函数参数也可以使用解构赋值

运用场景

字符串 常用

  1. string.include(‘he’) // 校验指定字符串是否包含

  2. string.startWith(‘h’) // 是否是以指定字符开头

  3. string.endWith(‘h’)

  4. const str = 自动换行 空格也ok

函数默认值

  1. 除了解构赋值的默认值
  2. 函数参数也支持默认值
  3. 一般情况下，如果函数需要默认值，也是函数的尾参数
  4. 当你的函数只需要一个参数具有默认值的时候，则可以把该参数定位为尾参数
  5. 当你的函数需要多个默认值的时候，则可以使用解构赋值的方式

rest

rest 表示剩余的意思，是将用户的传递过来的剩余参数解析放到一个数组中，如果没有剩余参数，...args也是一个空数组 ；rest 参数必须是最后一个，后面不能继续跟参数了

箭头函数 gose to

语法： 1.一个参数可以省略（）； 2.没有参数不能省略（）；3.箭头函数内部不要使用arguments（伪数组）， ...args可代替(数组)； 4.

  1. 箭头函数不能用在函数声明
  2. 只有函数表达式和匿名函数可以使用箭头函数
  3. 建议：所有需要匿名函数的地方都使用箭头函数，因为函数表达式用的很少，so 使用最多的就是匿名函数

任何箭头函数都会默认自动的 .bind(this); 手动 bind 无效

express 模块

返回值：即是函数 也是对象

用法:

app.use();

  1. 参数1表示：虚拟路径。这个路径只是判断用户请求的标识，当用户用此虚拟路径时，服务器才认为用户请求的是静态资源!

  2. 参数2表示：用户请求资源的路径

express.static()

  1. 返回值也是一个 方法， 这个被返回的方法就是根据用户请求去读取的目录下静态资源。

` 补充：

 1. 当有两个以上 app.use() 时; 虚拟路径不同时 都是可以读取静态资源的 不会出现覆盖的情况

 2. 当虚拟目录 名称相同时, 路径不同 ，系统会按照设置的先后顺序执行，如果没找到会去第二个设置的目录下去查找

 3. 第一个参数 虚拟路径不传时  默认的是 '/'


`

app.use() 和 app.verb() 的区别

app.verb() 有 app.get \ post \ put \ delete …

  1. 当通过 app.get() 注册路由的时候，用户只有 请求的路径和注册的路径完全一致时才能匹配到路由

  2. 当通过 app.use() 注册路由， 只要用户请求的路径是 给定的路径 或者 是给定路径下的某个路径都可以匹配到

app.alla() 和 app.use()

  1. app.all (‘/abc’),这个表示无论什么请求方式,只要请求路径匹配就可以触发该方法的回调函数

  2. app.use(‘/abc’),无论什么请求方式，只要路径是 由 /abc 或 /abc/xxx 或 /abc/a/d/c 都可以触发该方法的回调函数

` 一般 注册路由 都是用 app.get() \ app.post() \ app.all() …

一般 在注册中间的时候使用 app.use()
`

Dirty-Check 脏检查机制，ng双向数据绑定的原理

在angular初始化的时候，会先创建模块对象，初始化该模块的时候，会在该模块所管辖的view范围内，搜索所有的ng指令，找到指令之后，为每一个指令绑定一个$watcher，这个watcher会在数据或者dom元素发生改变的时候，被调用！

每当有一个watcher被触发，那么就会开启一次新的$digest，$digest中，会对所有的$watcher，一次执行来检测是否有数据变化，如果有变化，会重新启动一次$degist,依次循环，直到检测不到任何的变化位置，最终开始更新dom操作！

$apply
其实就是将相应的操作做了一个包装，先执行这些操作，然后再去调用$digest开始在脏检查

在angular中angular负责的所有的内容，都会默认的被$apply进行包裹，如果使用到了非angular控制的内容，比如setTimeout，在这里面的数据操作，将不会触发watcher，也就不会进行脏检查，所以，我们需要手动进行脏检查

使用步骤：
1.调用$apply来执行数据操作
$scope.$apply(function(){
    //做需要的数据操作
})
2.做完数据操作之后，手动调$digest方法
$scope.$digest();

angular-route

使用步骤：

1. 引入angular-route.js文件
2. 在使用该angular-route模板的模块中加入依赖项 ngRoute
3. 在模块的config方法中对angular-route进行配置  配置需要注入的内容为 $routeProvider
4. 使用when方法进行配置
angular
    .module("app",["ngRoute"])
    .config(["$routeProvider",function($routeProvider){
        $routeProvider.when("hash路径匹配规则",{
            template:"视图html代码",
            //template 属性和templateUrl只需要并且必须存在一个！
            templateUrl："视图路径",
            controller:"控制器"
        })
        //路由传参 在路由规则当中指定传参的形式，以及参数的名称
        .when("hash路径/:参数名称",{
            template:"视图html代码",
            templateUrl:"视图路径",
            controller:"控制器"
        })
        .otherwise({
            //这里做的就是当所有的匹配规则都不满足的情况下做的事情

            //redirectTo:就是进行路径跳转的操作
            redirectTo:"hash路径"
        })
    }])

    //获取路由传递进来的参数
    .controller("c",["$route","$routeParams",function($route,$routeParams){
        //1.通过$route服务获取
        $route.current.pathParams;
        //2.或通过$routeParams服务获取
        $routeParams;
    }])

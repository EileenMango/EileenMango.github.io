ES6

let

  1. let 的核心是加入块级作用域，{} 就是一个快
  2. 变量声明，必须先声明后使用，不允许重复声明；

const

  1. 声明常量，一旦声明，不可改变,否则报错；
  2. 普通数据类型不能修改，引用类型的数据可以修改属性值
  3. 声明时必须赋值，也是块级作用域，没有变量提升，不可重复声明；

总结

  1. 凡是变量使用 let
  2. 凡是不变的都是使用 const
  • 如果你定义的成员，一次都没有改变过，使用 const
  • 如果你定义的成员，最少做了一次修改，则一定是let
  • 所有相对来说，const使用要多一些；
  • 最佳使用建议：全部 const，报错时才改为let

数组的解构赋值

数组的解构赋值是按照数据的顺序来解构的

对象的解构赋值

对象的解构赋值 是按照对象的 key 来解构的；当解构不成功的时候，默认赋值undefined

运用场景

函数参数也可以使用解构赋值

运用场景

字符串 常用

  1. string.include(‘he’) // 校验指定字符串是否包含

  2. string.startWith(‘h’) // 是否是以指定字符开头

  3. string.endWith(‘h’)

  4. const str = 自动换行 空格也ok

函数默认值

  1. 除了解构赋值的默认值
  2. 函数参数也支持默认值
  3. 一般情况下，如果函数需要默认值，也是函数的尾参数
  4. 当你的函数只需要一个参数具有默认值的时候，则可以把该参数定位为尾参数
  5. 当你的函数需要多个默认值的时候，则可以使用解构赋值的方式

rest

rest 表示剩余的意思，是将用户的传递过来的剩余参数解析放到一个数组中，如果没有剩余参数，...args也是一个空数组 ；rest 参数必须是最后一个，后面不能继续跟参数了

箭头函数 gose to

语法： 1.一个参数可以省略（）； 2.没有参数不能省略（）；3.箭头函数内部不要使用arguments（伪数组）， ...args可代替(数组)； 4.

  1. 箭头函数不能用在函数声明
  2. 只有函数表达式和匿名函数可以使用箭头函数
  3. 建议：所有需要匿名函数的地方都使用箭头函数，因为函数表达式用的很少，so 使用最多的就是匿名函数

任何箭头函数都会默认自动的 .bind(this); 手动 bind 无效

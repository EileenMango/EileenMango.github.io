express 模块

返回值：即是函数 也是对象

用法:

app.use();

  1. 参数1表示：虚拟路径。这个路径只是判断用户请求的标识，当用户用此虚拟路径时，服务器才认为用户请求的是静态资源!

  2. 参数2表示：用户请求资源的路径

express.static()

  1. 返回值也是一个 方法， 这个被返回的方法就是根据用户请求去读取的目录下静态资源。

` 补充：

 1. 当有两个以上 app.use() 时; 虚拟路径不同时 都是可以读取静态资源的 不会出现覆盖的情况

 2. 当虚拟目录 名称相同时, 路径不同 ，系统会按照设置的先后顺序执行，如果没找到会去第二个设置的目录下去查找

 3. 第一个参数 虚拟路径不传时  默认的是 '/'


`

app.use() 和 app.verb() 的区别

app.verb() 有 app.get \ post \ put \ delete …

  1. 当通过 app.get() 注册路由的时候，用户只有 请求的路径和注册的路径完全一致时才能匹配到路由

  2. 当通过 app.use() 注册路由， 只要用户请求的路径是 给定的路径 或者 是给定路径下的某个路径都可以匹配到

app.alla() 和 app.use()

  1. app.all (‘/abc’),这个表示无论什么请求方式,只要请求路径匹配就可以触发该方法的回调函数

  2. app.use(‘/abc’),无论什么请求方式，只要路径是 由 /abc 或 /abc/xxx 或 /abc/a/d/c 都可以触发该方法的回调函数

` 一般 注册路由 都是用 app.get() \ app.post() \ app.all() …

一般 在注册中间的时候使用 app.use()
`
